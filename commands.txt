===========================================
 Attendance Management System - Setup Commands
===========================================


--- 1. Install Dependencies ---
pnpm install

--- 2. Setup Environment Variables ---
cp .env.example .env.local
# Then fill in your actual values in .env.local

--- 3. Setup Database (NeonDB) ---
# Create a NeonDB project at https://neon.tech
# Copy the connection string to DATABASE_URL in .env.local

# Generate database migrations
pnpm db:generate

# Push schema to database
pnpm db:push

# Open Drizzle Studio (database GUI)
pnpm db:studio

--- 4. Setup Clerk Authentication ---
# Create a Clerk project at https://clerk.com
# Copy the publishable key and secret key to .env.local
# Enable webhook in Clerk Dashboard:
#   - Go to Webhooks > Add Endpoint
#   - URL: https://your-domain.com/api/webhooks/clerk
#   - Events: user.created, user.updated, user.deleted
#   - Copy the signing secret to CLERK_WEBHOOK_SECRET

--- 5. Setup UploadThing ---
# Create an UploadThing project at https://uploadthing.com
# Copy the token to .env.local

--- 6. Setup Pusher ---
# Create a Pusher Channels app at https://pusher.com
# Copy the app_id, key, secret, and cluster to .env.local

--- 7. Setup QStash (for cron jobs) ---
# Create an Upstash account at https://upstash.com
# Go to QStash and create a token
# Copy the token and signing keys to .env.local
# Create a scheduled job:
#   - URL: https://your-domain.com/api/cron/end-sessions
#   - Schedule: every 1 minute (*/1 * * * *)

--- 8. Download face-api.js Models ---
# Models are included in public/models/ directory
# If missing, download from:
# https://github.com/justadudewhohacks/face-api.js/tree/master/weights
# Required models:
#   - ssd_mobilenetv1_model-weights_manifest.json + shard files
#   - face_landmark_68_model-weights_manifest.json + shard files
#   - face_recognition_model-weights_manifest.json + shard files

--- 9. Run Development Server ---
pnpm dev

--- 10. Build for Production ---
pnpm build
pnpm start

--- Other Useful Commands ---
pnpm lint          # Run ESLint
pnpm db:generate   # Generate new migration after schema changes
pnpm db:push       # Push schema changes directly (dev only)
pnpm db:studio     # Open Drizzle Studio GUI
